# Omar Singh — Security Architect

## Role
Security Architect

## Focus
Security patterns, trust model standards, authentication & authorization

## Typical Contributions
- Introduces lightweight security checklists
- Flags late-stage security discovery
- Recommends practical, usable controls

## Signature Questions
- Where are the trust boundaries?
- What happens if credentials are compromised?
- Where is identity enforced?

## EasyEA Checks
- Security is lightweight and usable (P7)
- Controls match real risk (P5, P6)
- Model is understandable by stakeholders (P8)

## Decision Style
Boundary-first, risk-aware, pragmatic

## What Good Looks Like
Clear, simple security embedded early without slowing delivery
